# トラブル対応

よくあるトラブルをここにまとめます。

## ①走行中にM5Stackの電源が落ちる
ミニCANユニットと中間ハーネスの接続部（ネジ固定部）が緩んでいる場合が考えられます。今一度、締め直しをお願いします。

## ②走行中にM5Stackの電源が落ちて、メータに警告灯が表示される。
①が起こると、ミニCANユニットから車両のCANバスに意図しない操作が行われる場合があります。具体的には、ミニCANユニットが送信状態で固着し、他の通信ができない状態になります。この状態を防ぐために、ミニCANユニットとM5Stack間のGroveケーブルのTXラインを切除しておく必要があります。（切除方法はセットアップページに記入済みです。）Kuruma-Loggerのファームウェアでは車両のCANバスに値を送信することはしていませんので、このような対応をしています。

2026/5/8 現在、ユーザの皆様にこのような対応をしていただく必要がないよう、Kuruma-Logger の推奨 CAN トランシーバをミニ CAN ユニットから PWRCAN モジュールに変更することを計画中です。すでに最新ファームでは対応できていますが、中間ハーネスの対応ができていません。自作していただければ PWRCAN モジュールでも動く状態です。PWRCAN 対応の中間ハーネスが完成次第、推奨ハードを変更します。

## ③GPSがNoDATAと表示される
まず OTA で v1.0.15 以降へ更新してください。v1.0.14 では、電源再投入後に GPS 受信機の設定が失われ、GPS が NoDATA になる場合がありました。v1.0.15 以降では、OTA 更新後に自動回復します。

更新後も改善しない場合は、下記を順に確認してください。

1. GNSS ユニットの DIP スイッチが `PS:3` / `TX:3` / `RX:2` ON になっているか確認してください。手順は [セットアップの DIP スイッチ設定](../setup/#ハードウェアの組み立て) を確認してください。
2. 屋外の空が開けた場所で 5〜10 分待機してください。初回測位や長時間未使用後は、測位まで時間がかかることがあります。
3. Tab5 を使用している場合は、GNSS の配線が M-Bus pin 21/22 になっているか確認してください。v1.0.13 以前の配線のままでは GPS を受信できません。
4. それでも解決しない場合は、[WEB インストーラ](https://app.kuruma-logger.com/setup/) から再インストールし、状況を [問い合わせフォーム](../contact/) から送信してください。再インストール後は [初回起動までの手順](../setup/#初回起動までの手順) に戻って動作を確認してください。

## ④マイページにログインできない
X などのアプリ内ブラウザからマイページや WEB インストーラを直接開くと、Google ログインの画面遷移や認証情報の保存が制限され、エラーになる場合があります。

下記の環境で開き直してください。

1. マイページへのログインは Safari / Chrome / Edge などの通常ブラウザで行ってください。
2. X の投稿から開いた場合は、メニューから「ブラウザで開く」「Safari で開く」「Chrome で開く」などを選んで開き直してください。
3. PC からファームウェアを書き込む場合は、USB 接続に対応した Chrome / Edge などのブラウザを使用してください。スマホのアプリ内ブラウザからは書き込みできません。
4. それでもログインできない場合は、使用している端末、ブラウザ名、表示されたエラー内容を [問い合わせフォーム](../contact/) から送信してください。

## ⑤Tab5でスマホがWi-Fi APに接続完了しない
M5Stack Tab5 で "Kuruma-Logger" AP は見えるが、iPhone/スマホ側が「接続中」のまま完了しない場合は、まず v1.0.18 以降へ更新してください。

v1.0.18 では、電源再投入や USB 抜き差し後の AP 起動、STA 接続失敗時の AP 維持、iPhone の接続確認応答を改善しています。

1. [WEB インストーラ](https://app.kuruma-logger.com/setup/) から Tab5 用の最新版をインストールしてください。インストール後は [初回起動までの手順](../setup/#初回起動までの手順) に戻って動作を確認してください。
2. インストール後、車両電源で Tab5 を起動し直してください。
3. AP は見えるが接続完了しない状態が続く場合は、スマホの Wi-Fi 設定から "Kuruma-Logger" を一度削除し、再接続してください。
4. それでも解決しない場合は、使用しているスマホ機種、iOS/Android バージョン、Tab5 の表示状態を [問い合わせフォーム](../contact/) から送信してください。

## 問い合わせ前に準備する情報

- 車種（ND / FL5）
- ファームウェアバージョン
- 発生手順（再現する操作）
- ログやスクリーンショットの有無

準備できたら [問い合わせフォーム](../contact/) から送信してください。
